Discord Events

Events are how your bot reacts to Discord. Tsundere keeps the Discord.js-style listener shape, then adds event autocomplete, intent hints, typed parameters, and hover docs.

Basic Pattern

client.on("messageCreate", async (message) => {
  if (message.author.bot) {
    return
  }

  if (message.content == "!ping") {
    await message.reply("pong")
  }
})

Event Reference

ready

Fires once when the gateway session is ready and the bot user is available. Use `client.once` for startup logs and initialization.

Parameters: none. Intents: none beyond normal gateway connection.

client.once("ready", () => {
  log(`Online as ${client.user.tag}`)
})

interactionCreate

Fires for slash commands, buttons, selects, modals, context menus, and autocomplete. Use interaction narrowing helpers before accessing specific fields.

Parameters: `interaction`. Intents: usually `Guilds` for guild interactions.

client.on("interactionCreate", async (interaction) => {
  if (interaction.isCommand("ping")) {
    await interaction.reply({ content: "pong", ephemeral: true })
  }
})

messageCreate

Fires when a message is created in a channel the bot can see. Reading `message.content` normally requires the Message Content intent.

Parameters: `message`. Intents: `GuildMessages`, often `MessageContent`.

messageUpdate and messageDelete

Use these for logs, cache-aware message tracking, and message state changes. Partial messages may need fetching before all fields are available.

Parameters: old/new message or deleted message. Intents: `GuildMessages`.

guildCreate and guildDelete

Fires when the bot joins or leaves a guild. Good for setup, analytics, and clearing guild-specific cache.

Parameters: `guild`. Intents: `Guilds`.

guildMemberAdd and guildMemberRemove

Fires when members join or leave. Useful for welcome messages, audit trails, and syncing external state. Member events require privileged member intent in many bots.

Parameters: `member`. Intents: `GuildMembers`.

guildBanAdd and guildBanRemove

Fires when users are banned or unbanned. Pair with audit log helpers when you need moderator attribution.

Parameters: `ban`. Intents: `GuildModeration` where available.

channelCreate, channelUpdate, channelDelete

Fires when guild channels change. Use these to maintain channel caches and validate bot permissions after structure changes.

Parameters: `channel`. Intents: `Guilds`.

roleCreate, roleUpdate, roleDelete

Fires when roles change. Good for permission caches, role helper validation, and audit tools.

Parameters: `role`. Intents: `Guilds`.

voiceStateUpdate

Fires when a member joins, leaves, mutes, deafens, streams, or moves voice channels. Compare old and new voice state to understand the change.

Parameters: `oldState`, `newState`. Intents: `GuildVoiceStates`.

threadCreate, threadUpdate, threadDelete

Fires for thread lifecycle changes. Use for forum-style bots, support flows, and cache maintenance.

Parameters: `thread`. Intents: `Guilds` and channel visibility permissions.

error, warn, debug

Operational events from the client. Keep these handlers small and send details to your logger or observability tool.

Parameters: error/message/debug info. Intents: none.

shardReady, shardDisconnect, shardReconnecting

Shard lifecycle events. Use them to track large bot availability and reconnect health.

Parameters vary by event. Intents depend on your shard configuration.
