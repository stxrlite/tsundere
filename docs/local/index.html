TypeScript familiarity. Discord-first tooling.

Build Discord bots with `.yuri` files and a real Node runtime.

Tsundere transpiles clean TypeScript-style `.yuri` code into JavaScript or TypeScript, installs a local `@tsundere/discord` runtime, and gives you one command for building, starting, developing, syncing commands, and opening docs.

Install

Use the release installer or install the CLI tarball globally. The installer opens these docs after setup.

Create

Create a Discord starter with a local runtime dependency and command discovery config already wired.

Run

Use `tsundere dev` while building and `tsundere start` in production. No manual `node build/main.ts` step.

Quick Start

tsundere create my-bot --template discord
cd my-bot
tsundere install
tsundere dev

For an existing project that cannot resolve `@tsundere/discord`, run `tsundere runtime install` and then `tsundere install` from the project folder.

Core Ideas

.yuri syntax: familiar TypeScript-style code with Discord-focused compiler checks and editor support.

Local runtime: Tsundere installs `@tsundere/discord` into `.tsundere/runtime/discord` so Node can resolve it without waiting on a public npm package.

Optimized launcher: TypeScript output is converted into runnable ESM JavaScript under `.tsundere/runtime-build`, then launched by `tsundere start` or `tsundere dev`.

Command discovery: command files under `src/commands` can be discovered automatically and converted into slash command metadata.

Version-aware updates: `tsundere version` prints the installed CLI version, and `tsundere updater` checks the configured GitHub releases feed.

Docs-first examples: sample `.yuri` files are kept in `docs/examples` so they remain close to the guides that explain them.

Light and dark docs: local docs follow system preference and include a mode toggle for manual switching.
