Discord Intelligence

Discord failures often come from missing intents, permissions, scopes, hierarchy position, deprecated APIs, or changed payloads. Tsundere should surface those problems before deployment.

Intent Analyzer

Analyze events like `messageCreate`, `guildMemberAdd`, and `presenceUpdate` and warn when required intents such as `GuildMessages`, `MessageContent`, `GuildMembers`, or `GuildPresences` are missing.

Permission Intelligence

Hover and diagnostics for operations like ban, timeout, channel create, role edit, and audit logs with required bot permissions, user permissions, hierarchy notes, and runtime risks.

Compatibility Checker

`tsundere doctor` should report Discord API version, wrapper version, package version, deprecated APIs, removed endpoints, changed intents, and migration suggestions.

Dashboards

VS Code panels should summarize required intents, missing permissions, OAuth scopes, and compatibility warnings as code changes.

Example Diagnostic

guild.members.ban(user)

Required Bot Permissions:
- BanMembers

Required Intent:
- GuildMembers

Hierarchy Requirements:
- Bot role must be above target member
- Invoking user must be above target member

Potential Runtime Errors:
- Missing Permissions
- Missing Access
- Unknown Member
