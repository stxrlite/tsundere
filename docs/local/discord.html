Discord Guide

Tsundere is not a bot framework. It gives you typed Discord API bindings, builders, interaction helpers, command registration utilities, diagnostics, and editor intelligence.

Tsundere Is Not a 1:1 Copy

You can write familiar imports and async code, but Tsundere projects are meant to use cleaner Discord layouts: small `main.yuri` files, command discovery, event files, typed builders, and interaction helpers. Read the layouts page when moving from a raw Discord.js bot.

Client Setup

import { Client, Intents } from "@tsundere/discord"

const client = new Client({
  token: env.DISCORD_TOKEN,
  intents: [
    Intents.Guilds,
    Intents.GuildMessages,
    Intents.MessageContent
  ]
})

client.login()

Events

The extension suggests Discord event names when you type `client.on("`. Selecting an event should provide the callback shape and hover documentation.

client.on("messageCreate", async (message) => {
  if (message.author.bot) {
    return
  }

  if (message.content == "!ping") {
    await message.reply("pong")
  }
})

Slash Commands

import { Slash } from "@tsundere/discord"

export default Slash.command("avatar")
  .description("Show a user's avatar")
  .option("user", "target", "User to inspect", { required: false })

With command discovery enabled, files in `src/commands` are indexed by `tsundere dev` and `tsundere commands sync`.

Embeds

const card = Embed.create()
  .title("Status")
  .description("The bot is online")
  .color("#ff7ab6")
  .timestamp()
  .toJSON()

Tsundere diagnostics warn about Discord limits such as long embed titles, long descriptions, custom ID limits, and common intent mistakes.

Typed Components

import { Component, Button } from "@tsundere/discord"

const BanButton = Component.define<{
  userId: string
}>("ban")

const button = Button.danger(
  BanButton.create({ userId: member.id })
).label("Ban")
