Updates

This section explains what changed in the current local build and how to apply it to projects.

Current update

Runtime, installer, docs, and local Discord package

  • Added a bundled `@tsundere/discord` package so projects can resolve the official Tsundere Discord wrapper locally.
  • Added `tsundere runtime install` to repair existing projects and write `.tsundere/runtime/discord`.
  • Changed `tsundere install` so it installs the local Discord runtime and falls back to npm when pnpm is missing.
  • Added a runnable Node runtime output folder at `.tsundere/runtime-build`.
  • Changed `tsundere start` so it runs through the Tsundere runtime instead of asking users to run Node manually.
  • Changed `tsundere dev` so it builds, runs, watches `.yuri` files, and restarts the runtime after changes.
  • Added these local HTML docs and made the installer open them after installation.
  • Added light and dark mode support for local docs.
  • Moved loose example files into `docs/examples` so the repo root stays clean.
  • Removed unused Markdown planning docs from the docs folder.
  • Added `tsundere version` and GitHub release checking through `tsundere updater`.
  • Updated the release builder so the release folder includes `tsundere-cli.tgz`, `tsundere-discord.tgz`, the VSIX, and installer scripts.

How to Update a Project

tsundere runtime install
tsundere install
tsundere build
tsundere start

How to Build a Release

npm run dist:release

Send the generated `release` folder to testers. They can run `installer.bat` or `install-tsundere.ps1`.

Planned Update Flow

Future versions should let `tsundere updater self` download the newest GitHub release, verify the package, install the CLI, refresh the VS Code extension, and reopen the local docs.

The current release checker is intentionally conservative. It tells you what version is available and where to download it without replacing files automatically.
