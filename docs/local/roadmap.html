Roadmap

Tsundere is meant to become a serious Discord-first language ecosystem without becoming a locked-in bot framework.

Compiler

  • Full AST parser for `.yuri` files.
  • Cleaner TypeScript AST emission.
  • Incremental build cache.
  • Better source maps from `.yuri` to runtime JavaScript.
  • More precise strict diagnostics.

Discord

  • Richer interaction narrowing.
  • Typed component payload serialization.
  • Schema-powered command options.
  • Better command sync for guild and global commands.
  • More Discord limit checks at compile time.

Editor

  • YuriLS workspace indexing.
  • Generated IntelliSense from installed Discord packages.
  • Auto imports for `@tsundere/discord` symbols.
  • Hover docs with required intents and permissions.
  • Command and component snippets.

Distribution

  • GitHub release updater.
  • Windows installer.
  • Published npm packages.
  • Extension marketplace release.
  • Versioned local docs.

Future Ideas

Less Setup, More Building

A new bot should feel ready in minutes: create the project, add a token, write commands, and run. No deploy-command script maze, no mystery loader folder, no fragile custom ID parsing.

Discord Knowledge in the Tooling

Tsundere should know Discord limits before runtime: embed sizes, modal inputs, button rows, required intents, permissions, option types, and stale command state.

Typed Interactions Without Lock-In

The goal is typed buttons, selects, modals, routes, and command options while still letting developers own the actual bot logic.

A Runtime That Feels Real

Build, start, dev, docs, package updates, version checks, command sync, and diagnostics should all live behind one clean CLI.
