Roadmap

Tsundere is a Discord-first language ecosystem. The long-term goal is not only cleaner syntax, but better protection, deeper static analysis, visual tooling, and a plugin ecosystem that solves real bot development pain.

Tsundere Protect

A modular Node.js protection pipeline with profiles for standard, advanced, and maximum builds. Protect should combine obfuscation, string encryption, runtime integrity, build randomization, and fingerprinting.

Read Protect docs

Discord Intelligence

Compiler and editor analysis for intents, permissions, role hierarchy, OAuth scopes, Discord API compatibility, deprecations, and migration warnings before deployment.

Read Intelligence docs

Command Visualizer

A VS Code panel that graphs commands, groups, subcommands, components, modals, handlers, services, and duplicate or dead commands.

Read Visualizer docs

Plugin Marketplace

A future plugin ecosystem for framework, database, Discord, compiler, and language plugins without bloating the core runtime.

Read Plugin docs

Future Ideas

Less Setup, More Building

A new bot should feel ready in minutes: create the project, add a token, write commands, and run. No deploy-command script maze, no mystery loader folder, no fragile custom ID parsing.

Discord Knowledge in the Tooling

Tsundere should know Discord limits before runtime: embed sizes, modal inputs, button rows, required intents, permissions, option types, stale command state, and deprecated APIs.

Commercial-Grade Builds

Protected builds should be randomized, fingerprinted, and traceable. The goal is accountability and friction against casual reverse engineering, not impossible security promises.

A Runtime That Feels Real

Build, start, dev, docs, package updates, version checks, command sync, doctor reports, and diagnostics should all live behind one clean CLI.
