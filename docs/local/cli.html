CLI and Runtime

The CLI is the normal entry point for Tsundere projects. You should not need to run `node build/main.ts` manually.

Runtime Flow

`tsundere build` compiles `.yuri` files into `build/` and emits runnable JavaScript into `.tsundere/runtime-build/`.

`tsundere start` runs `.tsundere/runtime-build/main.js`. If runtime output is missing, it builds first.

`tsundere dev` builds, syncs command metadata, starts the runtime, watches `.yuri` files, and restarts the runtime after changes.

tsundere startRun the compiled app through the Tsundere runtime.
tsundere fingerprint inspectInspect protected build metadata or fingerprints inside an output file.
tsundere versionPrint the installed Tsundere version.
tsundere updaterCheck the configured GitHub releases feed for CLI updates.
tsundere commands syncGenerate command discovery metadata.
tsundere types syncGenerate Discord IntelliSense metadata.
tsundere plugin install user/repoInstall official plugins, npm packages, local plugins, or GitHub plugin links.
tsundere docsOpen these local docs.
