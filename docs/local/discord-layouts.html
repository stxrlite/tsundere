Discord Layouts

Tsundere keeps familiar JavaScript-shaped expressions, but Discord projects are organized around clean files, command discovery, typed builders, and interaction helpers. You do not need a giant manual command loader.

Recommended Project Layout

src/
  main.yuri
  events/
    ready.yuri
    interactionCreate.yuri
    guildMemberAdd.yuri
  commands/
    ping.yuri
    avatar.yuri
    admin/
      ban.yuri
  components/
    confirmButton.yuri
  modals/
    reportModal.yuri
  services/
    users.yuri
    github.yuri

Main File

Keep `main.yuri` small. Create the client, register shared event handlers, then login.

import { Client, Intents } from "@tsundere/discord"
import { registerReadyEvent } from "./events/ready"
import { registerInteractionEvents } from "./events/interactionCreate"

const client = new Client({
  token: env.DISCORD_TOKEN,
  intents: [
    Intents.Guilds,
    Intents.GuildMembers,
    Intents.GuildMessages,
    Intents.MessageContent
  ]
})

registerReadyEvent(client)
registerInteractionEvents(client)

client.login()

Event Files

Events are normal code. YuriLS should autocomplete event names, callback params, required intents, and common mistakes.

export function registerReadyEvent(client) {
  client.once("ready", () => {
    log(`Online as ${client.user.tag}`)
  })
}

Command Files

Command files export command definitions. `tsundere dev` and `tsundere commands sync` discover them from `src/commands`.

import { Slash } from "@tsundere/discord"

export default Slash.command("avatar")
  .description("Show a user's avatar")
  .option("user", "target", "User to inspect", { required: false })

Route-Based Commands

Route-based commands are configurable. You can make folders become command groups, or disable that and use names from each command file.

{
  "commands": {
    "discovery": true,
    "routeBased": false,
    "directory": "./src/commands",
    "groups": {
      "./src/commands/admin": {
        "routeBased": true,
        "groupName": "admin"
      }
    }
  }
}

Interaction Router Style

You can still use raw `interactionCreate`, but larger bots should route commands, components, modals, and selects into separate handlers.

client.on("interactionCreate", async (interaction) => {
  if interaction.isCommand("ping") {
    await pingHandler(interaction)
  }

  if interaction.isButton("confirm") {
    await confirmHandler(interaction)
  }

  if interaction.isModal("report") {
    await reportHandler(interaction)
  }
})
